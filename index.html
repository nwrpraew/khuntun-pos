🌐 เชื่อมต่อข้ามเครื่อง (Firebase)

ลูกค้าสแกน QR เปิดเมนูบนมือถือตัวเอง → กดส่งออเดอร์ → ขึ้นในคิวร้านอัตโนมัติ (ต้องตั้ง Firebase Realtime Database ฟรีจาก Google)

วิธีตั้งค่า Firebase (~5 นาที ครั้งเดียว):
  1. เข้า console.firebase.google.com ล็อกอินด้วย Google
  2. กด Add project → ตั้งชื่อ (เช่น khun-tun-pos) → ปิด Analytics → Create
  3. ใน sidebar กด Build → Realtime Database → Create Database
  4. เลือก location Singapore (asia-southeast1) → Next
  5. เลือก Start in test mode → Enable
  6. คลิกที่ tab Rules วาง:
    { "rules": { ".read": true, ".write": true } }→ Publish
  7. กลับไปที่ Project settings (รูปเฟือง ⚙️) → scroll ลง "Your apps" → กด </> (Web) → ตั้งชื่อ → Register
  8. Copy ค่า apiKey และ databaseURL จาก firebaseConfig ที่แสดงขึ้นมา → วางในช่องด้านบน → กด "เชื่อมต่อ"

🟢 แจ้งเตือนผ่าน LINE OA

ส่งแจ้งเตือนเข้า LINE เมื่อมีออเดอร์ใหม่ / เปลี่ยนสถานะ / ชำระเงิน (ดูวิธีตั้งค่าในไฟล์ LINE_SETUP.md)

หมายเหตุ: ลูกค้าจะเห็น QR + รหัสออเดอร์หลังกดส่ง — ถ้าแอด LINE OA แล้วพิมพ์รหัสในแชต จะได้รับการแจ้งเตือนสถานะ

🔔 แจ้งเตือนออเดอร์ใหม่

เล่นเสียง "ติ๊ง ติ๊ง" เมื่อมีออเดอร์ใหม่จากลูกค้าเข้ามา

หมายเหตุ: เบราว์เซอร์ต้องให้กดอะไรในหน้าก่อน 1 ครั้ง เสียงถึงจะเล่นได้

QR สำหรับลูกค้า

สร้าง QR เพื่อให้ลูกค้าสแกนเปิดเมนู (เชื่อม Firebase แล้วใช้ข้ามเครื่องได้)

การจัดการข้อมูล

วิธีใช้งาน

  1. เพิ่ม วัตถุดิบ + เมนู (พร้อมสูตร)
  2. ใช้แท็บ ขาย/POS แตะเมนูสร้างออเดอร์ หรือสแกน QR ลูกค้า
  3. ยืนยันออเดอร์ → ระบบหักสต็อกอัตโนมัติ
  4. ดู รายงาน ยอดขาย/กำไรรายวัน + กราฟ 7 วัน
  5. ใช้ "QR ลูกค้า" ให้ลูกค้าสั่งเอง → ออเดอร์เข้าคิวรอยืนยัน
